Flickr Widget

<div id="flickr_widget-1" class="widget flickr-widget clearfix">

    <h4 class="widget-title">Flickr Photostream</h4>
    
    <div id="flickr_widget" class="flickrfeed" data-id="613394@N22" data-count="9" data-type="group"></div>

</div>

Options:

  • data-id - Your Flickr User/Group ID, Get it Here
  • data-count - Number of Flickr Images to retrieve.
  • data-type - Type of Flickr ID, ie. user or group

Twitter oAuth Setup for API v1.1

The new Twitter API v1.1 no longer supports unauthenticated User Timeline calls so you'll have to now authenticate to OAuth so that you can show your Twitter Feeds on your Websites. Follow the instructions below to setup your authentication:

  1. Visit https://dev.twitter.com/apps/ and sign in using your Twitter username and password. This doesn't have to be the username or password of the stream you need access to, just a Twitter account you control.
  2. Select Create new application and enter the application details.
    1. The name and description can be anything you like really, but you can't use 'Twitter' in the name.
    2. The website field can be your main website and doesn't have to be the site where your Twitter feed or feeds are located.
    3. Callback URL can be left blank
  3. Enter the CAPTCHA info and click create
  4. On the next details screen, click Create my access token. You may need to refresh the page after a few seconds if it doesn't appear automatically.
  5. Make a note of the Consumer key, Consumer secret, Access token and Access token secret as highlighted below.

Follow these steps after you create your Twitter APP successfully as described above:

  1. Open the html/functions/twitter/tweets.php file.
  2. Enter your Twitter APP Details as mentioned below: 
    $consumerkey = ""; // Twitter App - Consumer Key for OAuth
$consumersecret = ""; // Twitter App - Consumer Secret for OAuth
$accesstoken = ""; // Twitter App - Access Token for OAuth
$accesstokensecret = ""; // Twitter App - Access Token Secret for OAuth
    and Save the file.

Note Remember to set the permissions of the html/functions/twitter/cache folder to 755 so that the Twitter Feed cache files can be saved otherwise your Twitter Feeds won't work.

Instagram Widget

<div class="widget flickr-widget clearfix">

    <h4 class="widget-title">Instagram Photos</h4>
    
    <div class="instagram" data-tag="dribbble" data-count="9" data-type="tag"></div>

</div>

Options:

  • data-user - Any valid Instagram Username
  • data-tag - Any valid Instagram Tag
  • data-count - Number of Instagram Images to retrieve.
  • data-type - Type of Query you are making, ie. user or tag

Note You'll first have to setup your Instagram clientID & accessToken in the html/js/custom.js under $.fn.spectragram.accessData. To get your Instagram clientID & accessToken, please visit Instagram API Website.

Dribbble Widget

<div class="widget flickr-widget clearfix">

    <h4 class="widget-title">Dribbble Shots</h4>
    
    <div class="dribbble" data-user="envato" data-count="9" data-type="user"></div>

</div>

Options:

  • data-user - Any valid Dribbble Username
  • data-list - Type of Dribbble List, ie. popular, everyone or debuts
  • data-count - Number of Dribbble Shots to retrieve.
  • data-type - Type of Query you are making, ie. user, follows or list
